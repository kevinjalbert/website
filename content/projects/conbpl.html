Description

This research project focuses on the development of a new approach for active testing concurrent Java programs. Our research aims to:

  • Detect possible concurrency bugs in a software system using bug patterns (based on previously detected bugs).
  • Combine our identification of possible bugs with existing tools to optimize the testing of concurrent software.

We use clone detection, a technique for finding duplicatedsource code, as the foundation for detecting bug patterns in concurrent software. If we can identify a bug pattern as being duplicated then we can target our regression testing effort to the exploration of interleavings relevant to the potential bug. Our implementation is built by modifying the ConQAT clone detection framework. ConQAT is capable of detecting exact, near-exact and gapped clones.
