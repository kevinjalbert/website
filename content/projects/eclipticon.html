Description

Eclipticon is a testing and development software tool to help users optimize for concurrency. This tool will be in the form of an Eclipse plugin and will be targeted to a specific group of users that is developers and testers of concurrent software. The goal of the tool is to ease difficulties that arise when using concurrency in the Java programming language.

The tool is called Eclipticon as it will combine concurrency utilities into the Eclipse IDE. Eclipticon will be an open‐source project that is modeled after the IBMs's closed‐source research testing tool, ConTest. There are some main features that separate these two projects from one another.

First off, Eclipticon will be open‐source as well as including a full‐featured interface with Eclipse. IBM's tool is closed‐source and does not provide a sufficient GUI component. ConTest has the ability to plugin to Eclipse, though it is hardly worth mentioning.

The main differentiating feature is Eclipticon will allow the user to specifically select areas of the code to test. The user can specify these instrumentation points either manually or automatically. The automatic generation of these test areas will be random, though following some user‐adjustable configurations. The testing will occur by inserting thread delays that will cause the newly instrumented code to explore a larger set of thread interleavings.
